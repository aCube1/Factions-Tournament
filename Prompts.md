# Prompts utilizados no projeto:

1. Análise do CRC e integração com o código- Desenvolvimento do código com base no cartão CRC.- Integração do CRC com código existente.
2. Integração de código existente- Análise da classe Arena existente.- Explicação de trechos específicos do código.
3. Interface de controle (IController)- Criação do IController seguindo a estrutura do projeto.- Discussão sobre métodos necessários e documentação.
4. Estrutura de Controllers e fluxo de turno- Coleta de comandos dos controllers.- Definição de responsabilidades entre PlayerController e AIController.- Local de implementação das mudanças.
5. Refatoração arquitetural- Separação entre decisão e execução.- Ajuste do IController para retorno de ações.- Impacto da nova arquitetura no jogo.
6. Sistema de Actions- Identificação das mudanças necessárias a partir do CRC.- Definição do local da interface Action.- Criação e organização das novas classes.
7. Estrutura de pastas- Comparação entre estrutura atual e estrutura sugerida.- Validação da organização por pacotes.
8. Implementação prática- Criação da primeira Action (AttackAction).- Definição dos próximos passos após a implementação.
9. Ajustes finais- Refatoração apenas de IController e Arena.- Manutenção dos Controllers para implementação futura.
10. Justificativa técnica- Explicação detalhada do propósito das mudanças no código.
11. Versionamento- Listagem do que foi implementado para commit organizado.
12. Debug this code, just tell me what might be wrong. DO NOT CHANGE ANYTHING!
13. Generate a Lanterna code to draw <Insert UI Element Here>.

> Observação Final
> Este documento resume os prompts técnicos que guiaram a evolução arquitetural do projeto, com foco em um modelo baseado em Actions e alinhado ao CRC.
